# 商品卡密资源绑定功能 - 实现说明

## 功能说明

已为商品管理模块添加了卡密资源绑定功能。管理员可以在商品添加/编辑页面：
- 搜索并选择卡密资源进行绑定
- 查看当前绑定的卡密资源
- 解除已绑定的卡密资源
- 绑定是可选的，可以不绑定

## 已完成的工作

### ✅ 后端部分（已完成）

1. **数据库迁移脚本**
   - 文件：`database/add_card_resource_binding.php`
   - 功能：为 products 表添加 card_resource_id 字段

2. **后端 API**
   - `GET /admin/products/card-resources` - 获取可用卡密资源列表（支持关键词搜索）
   - `POST /admin/products/:id/unbind-card-resource` - 解除绑定
   - 商品创建/更新接口已支持 `card_resource_id` 字段

3. **模型关联**
   - Product 模型添加了 cardResource 关联
   - 商品列表和详情接口返回 `card_resource_name`

### ⏳ 前端部分（待完成）

需要修改管理后台的商品添加/编辑页面，添加卡密资源选择器UI。

```bash

```

如果 PHP 不在 PATH 中，使用完整路径：
```bash
D:\phpstudy_pro\Extensions\php\php8.0.2nts\php.exe database/add_card_resource_binding.php
```

或者直接在数据库中执行 SQL：
```sql
ALTER TABLE `products` 
ADD COLUMN `card_resource_id` BIGINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '关联的卡密资源ID' 
AFTER `kind_category_id`;

ALTER TABLE `products` 
ADD INDEX `idx_card_resource_id` (`card_resource_id`);
```

## API 接口示例

### 搜索卡密资源
```javascript
// 请求
GET /admin/products/card-resources?keyword=Roblox

// 响应
{
  "code": 0,
  "msg": "success",
  "data": [
    {
      "id": 1,
      "title": "Roblox账号资源",
      "module_type": "account",
      "status": 1
    }
  ]
}
```

### 保存商品（带卡密资源绑定）
```javascript
// 请求
POST /admin/products
{
  "name": "Roblox 1000 Robux",
  "category_id": 1,
  "card_resource_id": 1,  // 绑定卡密资源ID
  // ... 其他字段
}

// 响应
{
  "code": 0,
  "msg": "创建成功",
  "data": { "id": 10 }
}
```

### 解除绑定
```javascript
// 请求
POST /admin/products/10/unbind-card-resource

// 响应
{
  "code": 0,
  "msg": "已解除绑定",
  "data": { "id": 10, "card_resource_id": 0 }
}
```

## 注意事项

1. **数据库迁移必须先执行**，否则后端接口会报错
2. 卡密资源的 `title` 字段已存在于数据库中，无需额外添加
3. 绑定是可选的，`card_resource_id` 为 0 表示未绑定
4. 只显示状态为启用（status=1）的卡密资源
5. 前端代码使用原生 JavaScript，兼容性好
