# 商品批量删除功能文档

## 功能概述
在商品管理页面添加了批量删除功能，允许管理员一次性删除多个商品。

## 实现细节

### 后端实现

#### API 端点
- **路由**: `POST /admin/products/delete-batch`
- **文件**: `app/admin/route/route.php` (第37行)

#### 控制器方法
- **文件**: `app/admin/controller/Product.php`
- **方法**: `deleteBatch()` (第207-244行)

**请求参数**:
```json
{
  "ids": [1, 2, 3, 4, 5]
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "成功删除 5 个商品",
  "data": {
    "count": 5
  }
}
```

**删除逻辑**:
1. 验证 `ids` 参数是否为有效数组
2. 过滤并转换为整数数组
3. 对每个商品ID执行：
   - 解除绑定的卡密：将 `xos_card.product_id` 设为 0
   - 解除绑定的卡密资源：将 `xos_card_resource.product_id` 设为 0，`is_common` 设为 1（变为通用资源）
   - 删除商品记录
4. 返回删除数量

### 前端实现

#### UI 组件
**文件**: `public/static/admin/js/card-admin.js`

1. **表格复选框** (第1547行)
   - 每行第一列添加复选框
   - 绑定 `data-product-id` 属性

2. **全选复选框** (第1606行)
   - 表头第一列添加全选复选框
   - ID: `selectAllProducts`

3. **批量删除按钮** (第1467行)
   - 工具栏中添加按钮
   - ID: `batchDeleteProductsBtn`
   - 默认隐藏 (`display: none`)
   - 选中商品后显示，并显示数量

#### 状态管理
```javascript
state.selectedProductIds = []  // 存储选中的商品ID数组
```

#### 核心函数

**1. updateBatchDeleteButton()** (第1676-1686行)
- 根据选中数量显示/隐藏批量删除按钮
- 更新按钮文本显示选中数量

**2. bindProductTableEvents()** (第1609-1648行)
- 绑定全选复选框事件
- 绑定单个复选框事件
- 同步全选状态

**3. 批量删除事件处理** (第1503-1522行)
- 验证是否选中商品
- 显示确认对话框
- 调用 API 删除
- 刷新商品列表

#### 用户交互流程

1. **选择商品**
   - 用户勾选单个商品复选框
   - 或点击表头全选复选框

2. **显示按钮**
   - 选中商品后，批量删除按钮自动显示
   - 按钮文本显示：`批量删除 (N)`

3. **执行删除**
   - 点击批量删除按钮
   - 弹出确认对话框：`确定要删除选中的 N 个商品吗？删除后会自动解除所有绑定的卡密和卡密资源。`
   - 确认后发送 API 请求
   - 删除成功后清空选中状态并刷新列表

## 安全特性

1. **二次确认**: 删除前弹出确认对话框，防止误操作
2. **自动解绑**: 删除商品前自动解除所有关联的卡密和卡密资源绑定
3. **参数验证**: 后端验证 `ids` 参数的有效性
4. **错误处理**: 前后端都有完善的错误提示

3. **UI 测试**

4. **数据完整性测试**
   - 验证删除后卡密的 `product_id` 是否设为 0
   - 验证删除后卡密资源是否变为通用资源
   - 验证商品记录是否正确删除
