# 客户端流程摘要

## 说明

- 这份文档只针对客户端 46 页。
- 目的不是重复截图，而是把页面按真实业务流程重新串起来，方便前端按模块开发。
- 页面详情、截图、热点坐标请继续配合 `README.md` 与 `client_pages.json` 一起使用。

## 1. 认证与登录流程

主页面：

- 01《登录》
- 02《微信号登录》
- 03《登录》
- 04《注册》
- 05《忘记密码》
- 06《登录-密码登录》

推荐前端实现：

- 将 01、02、03、06 视为同一个认证模块下的不同登录方式。
- 推荐拆成四个路由：短信登录、微信授权登录、密码登录、注册/重置密码。
- 登录成功后统一跳转首页，不需要把每个原型页当成独立业务实体。

建议流程：

1. 启动进入 01《登录》
2. 用户可选择微信授权或手机号登录
3. 手机号登录继续分为验证码登录和密码登录
4. 注册与忘记密码是认证分支页面
5. 所有登录成功统一进入 11《首页》

前端实现重点：

- 登录方式切换
- 协议勾选状态
- 获取验证码按钮倒计时
- 登录成功后的 token、用户信息初始化

## 2. 商城浏览与下单流程

主页面：

- 11《首页》
- 08《商城》
- 13《搜索》
- 07《首页-商品详情》
- 12《首页-商品详情 -下单》
- 39《首页-商品详情 -我要批发》
- 14《首页-药贴》
- 15《首页-云台活血贴》
- 46《首页-商品详情 Copy 1》
- 09《采购车》

建议理解方式：

- 11 是首页聚合页。
- 08 是商城列表页。
- 07、14、15、46 本质都属于商品详情页或详情变体。
- 12 是下单确认页。
- 39 是批发入口页。
- 09 是购物车页。

建议流程：

1. 首页或商城进入商品详情
2. 在商品详情中选择规格、数量
3. 用户可以立即购买、加入购物车、进入批发流程
4. 立即购买进入 12《下单》
5. 支付完成后进入订单页状态流转

前端实现重点：

- 首页与商城的商品卡片复用
- 商品详情的规格切换、价格展示、图文详情
- 底部操作栏：加入购物车、立即购买、我要批发
- 下单页的地址、商品、金额、提交按钮

## 3. 我的与普通订单流程

主页面：

- 10《我的》
- 16《我的-我的订单》
- 17《我的-我的订单-待付款》
- 18《我的-我的订单-待发货》
- 19《我的-我的订单-待收货》
- 20《我的-我的订单-待评价》
- 21《我的-我的订单-售后》
- 22《我的-我的订单-待发货-申请退款》
- 23《我的-我的订单-待发货-申请退款》
- 24《我的-我的订单-售后 -已寄回》
- 25《我的-我的订单-售后 -填写单号》
- 26《我的-我的订单-售后 -已退款》
- 27《我的-我的订单-待评价-交易成功》
- 28《我的-我的订单-待评价-评价》
- 37《我的-发票管理》
- 38《我的-地址管理》

推荐前端实现：

- 将 16 作为普通订单列表页。
- 17 到 28 视为订单详情/售后详情在不同状态下的页面快照。
- 推荐使用同一套订单详情组件，通过 `status` 和 `afterSaleStatus` 控制展示，不建议每页单独写死。

建议流程：

1. 10《我的》进入订单、OEM 订单、地址、发票等模块
2. 16《我的-我的订单》按 Tab 查看各订单状态
3. 订单在待付款、待发货、待收货、待评价之间流转
4. 售后链路从申请退款开始，延伸到填写单号、寄回、退款完成
5. 最终完成评价

前端实现重点：

- 订单状态 Tab
- 订单卡片与详情页复用
- 金额模块、物流模块、客户操作按钮
- 售后原因、售后金额、寄回物流单号
- 评价输入框、评分、图片上传占位

## 4. OEM 订单流程

主页面：

- 29《我的-OEM订单》
- 30《我的-OEM订单-待确认》
- 31《我的-OEM订单-样品记录》
- 32《我的-OEM订单-设计确认》
- 33《我的-OEM订单-合同协议》
- 34《我的-OEM订单-生产阶段》
- 35《我的-OEM订单-生产阶段 -待生产-2》
- 36《我的-OEM订单-已完成》
- 44《工厂OEM》
- 45《工厂OEM-1》

推荐前端实现：

- 29 作为 OEM 订单入口/列表页。
- 30 到 36 视为同一 OEM 订单详情在不同业务阶段的页面快照。
- 使用统一的 `OEMStepTimeline` 和 `OEMOrderDetail` 容器，按阶段渲染不同字段与按钮。

阶段建议：

1. 待确认
2. 样品记录
3. 设计确认
4. 合同协议
5. 生产阶段
6. 已完成

前端实现重点：

- OEM 订单状态时间线
- 客户信息、合作方式、包材、商标、批件等字段块
- 上传记录列表
- 提交审核、查看协议、查看设计稿等操作入口

## 6. 最适合的开发顺序

1. 认证页：01、03、04、05、06
2. 首页/商城/商品详情：11、08、07、12、09
3. 我的与基础资料：10、37、38
4. 普通订单：16 到 28
5. OEM 订单：29 到 36、44、45
6. 内容辅助页：40、41、42、43

## 7. 强烈建议的代码组织

- 商品详情做成单一详情页，通过 mock 数据切换不同商品
- 普通订单做成“列表 + 详情 + 售后子流程”
- OEM 做成“列表 + 阶段化详情”
- 不要按原型页数量一页一个独立组件硬编码，否则后期维护成本会很高
