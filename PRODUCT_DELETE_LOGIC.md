# 商品删除逻辑说明

## 修改内容

### 原逻辑（已移除）
- ❌ 检查商品下是否有卡密
- ❌ 如果有卡密，禁止删除，返回错误："该商品下存在卡密，禁止删除"

### 新逻辑（已实现）
删除商品时自动执行以下操作：

1. **解除卡密绑定**
   - 将所有绑定到该商品的卡密的 `product_id` 设置为 `0`
   - 这些卡密变为"未绑定商品"状态

2. **解除卡密资源绑定**
   - 将所有绑定到该商品的卡密资源（`is_common=0`）的 `product_id` 设置为 `0`
   - 同时将 `is_common` 设置为 `1`，变为"通用资源"

3. **删除商品**
   - 删除商品记录

## 代码实现

```php
public function delete(int $id): \think\Response
{
    $product = ProductModel::find($id);
    if ($product === null) {
        return $this->error('商品不存在', 404, 404);
    }

    // 1. 解除卡密绑定
    Card::where('product_id', $id)->update(['product_id' => 0]);
    
    // 2. 解除卡密资源绑定，变为通用资源
    \app\model\CardResource::where('product_id', $id)
        ->where('is_common', 0)
        ->update(['product_id' => 0, 'is_common' => 1]);

    // 3. 删除商品
    $product->delete();
    
    return $this->success([], '商品删除成功');
}
```

## 影响范围

### 卡密（Card）
- **删除前**：`product_id = 11`（绑定到商品11）
- **删除后**：`product_id = 0`（未绑定）
- **状态**：卡密仍然存在，只是不再绑定到任何商品

### 卡密资源（CardResource）
- **删除前**：`product_id = 11`, `is_common = 0`（指定商品）
- **删除后**：`product_id = 0`, `is_common = 1`（通用资源）
- **状态**：卡密资源仍然存在，变为通用资源，可以绑定到其他商品

## 优点

1. **用户体验好**：删除商品不会被阻止，操作流畅
2. **数据不丢失**：卡密和卡密资源都保留，只是解除绑定
3. **灵活性高**：解除绑定后的资源可以重新绑定到其他商品
4. **符合业务逻辑**：商品删除不应该影响卡密数据的存在

## 测试步骤

1. **创建测试商品**
   - 新增一个商品（例如：测试商品A）
   - 记录商品ID（例如：ID=11）

2. **绑定卡密**
   - 在卡密管理中，创建一些卡密并绑定到该商品
   - 或者在商品编辑页面绑定卡密资源

3. **删除商品**
   - 在商品列表中，点击"删除"按钮
   - ✅ 应该成功删除，不会提示"该商品下存在卡密，禁止删除"

4. **验证卡密状态**
   - 打开卡密管理页面
   - ✅ 之前绑定的卡密应该还存在
   - ✅ 这些卡密的"对应商品"应该显示为空或"未绑定"

5. **验证卡密资源状态**
   - 打开卡密管理页面（账号密码类/下载连接类/教程类）
   - ✅ 之前绑定的卡密资源应该还存在
   - ✅ 这些资源应该变为"通用资源"（is_common=1）

## 注意事项

1. **不可逆操作**：删除商品后无法恢复（除非有数据库备份）
2. **批量解绑**：如果商品下有大量卡密，解绑操作可能需要一些时间
3. **建议确认**：前端可以添加二次确认对话框，提示用户删除商品会解除所有绑定

## 可选改进

如果需要更友好的提示，可以在前端添加确认对话框：

```javascript
// 删除商品时的确认提示
if (confirm('删除商品会自动解除所有绑定的卡密和卡密资源，确定要删除吗？')) {
    // 执行删除
}
```
